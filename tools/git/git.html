Versiyon Kontrol Sistemi

Git (Version Control)

Analitik modellerinizi ve projelerinizi güvenle geliştirin, geçmişe dönün (time travel) ve diğer veri bilimcileriyle sorunsuz işbirliği (collaboration) yapın.

🌳 Git Branching & Commit History

Proje geçmişinizin adım adım kayıt altına alınması, branch ağacı ve merge (birleştirme) süreçleri.

📸 Snapshot Sistemi

Git, dosyalarınızı bir dizin gibi değil, verinizin o anki durumunun anlık görüntüleri (snapshot) olarak tutar. Hatalı bir analiz yaptığınızda kolayca bir önceki temiz sürüme dönebilirsiniz.

🌿 Branching (Dallanma)

Mevcut üretim kodunuzu (main/master) etkilemeden yeni makine öğrenmesi algoritmaları denemek için bağımsız çalışma dalları oluşturmanızı sağlar.

🌐 GitHub / GitLab Entegrasyonu

Yalnızca yerel bilgisayarınızda değil, kodunuzu bulutta güvenle saklamanızı, ekibinizle eşzamanlı çalışmanızı ve CI/CD süreçlerini yönetmenizi sağlar.

Git'in Artıları ve Eksileri

✅ Avantajları

  • ▪ Zaman Makinesi: Hatalı bir kod yazdığınızda, istediğiniz eski bir kopyaya saniyeler içinde dönebilirsiniz.
  • ▪ Dağıtık Çalışma: İnternet olmadan (offline) commit yapabilme ve tam bir repo kopyasını lokalde tutma imkanı sağlar.
  • ▪ Takım İşbirliği: Branch (dal) yapısı sayesinde projenin ana kodunu bozmadan aynı anda onlarca kişinin kod yazmasını kolaylaştırır.

⚠️ Dikkat Edilmesi Gerekenler

  • ▪ Karmaşık Öğrenme Eğrisi: Merge conflict, rebase ve reset gibi kavramları anlamak acemiler için korkutucu olabilir.
  • ▪ Büyük Veri Yönetimi: GB'larca büyüklükteki veri setleri veya görseller için tasarlanmamıştır (Git LFS gerektirir).
  • ▪ Komut Satırı Bağlılığı: Arayüzleri olsa da tam güç için terminal komutlarına aşina olmayı gerektirir.

💻 Git Syntax & Kullanım Örneği

Versiyon Kontrol

Günlük analiz projelerinizde terminal üzerinden versiyon yönetimi yapmak için temel komutlar. 

# Yeni bir Git deposu başlatmak
git init

# Değiştirilen dosyaları izleme (Staging area)
git add analysis.ipynb

# Değişiklikleri kayıt altına alma (Commit)
git commit -m "feat: xgboost modeli performansı artırıldı"

# Yeni model denemesi için branch (dal) oluşturma
git checkout -b experimental-model

# Değişiklikleri GitHub'a (veya uzak sunucuya) yükleme
git push origin main
